# Skill：PDF 首页高密度速览 + 证据型原文标注

## 1. 技能目标

对用户上传的论文 PDF 进行文献标注，生成新的已标注 PDF。

本技能包含两类标注：

1. **首页顶部高密度文献速览卡**  
   用于快速复盘论文：方法、研究对象、实验/数据、结论、关键结果、关键词、注意点。

2. **论文原文证据型标注**  
   在论文原文的实际出处位置，对关键概念和重要证据句进行少量、去重、有重点的标注。

核心原则：

> 首页文本框是“速览卡”，原文标注是“证据位置”。  
> 禁止简单按关键词全文重复标注。  
> 红线标关键概念，黄标标证据句。

---

## 2. 输入

用户通常会提供：

1. 一篇论文 PDF；
2. 可选：文献标注模板；
3. 可选：用户指定日期；
4. 可选：用户指定文件命名方式。

如果用户没有额外说明，则使用本技能默认规则。

---

## 3. 输出

输出一个新的 PDF 文件。

要求：

1. 不新增文献标注页；
2. 保留原 PDF 内容；
3. 在第一页顶部加入高密度文献速览文本框；
4. 在论文原文实际位置做少量证据型标注；
5. 红色下划线用于关键概念；
6. 黄色高亮用于证据句；
7. 输出文件名按照命名规则生成；
8. 暂时不使用“代号”。

---

## 4. 文件命名规则

暂时不使用代号。

命名格式：

```text
日期-论文名（关键词或主要内容，发表单位 作者，）_已标注.pdf
```

示例：

```text
20190702-Detection of Rust Defects Based on HSV Color Model（HSV+retinex+ICM，IEEE Ukraine Conference Iryna Ivasenko，）_已标注.pdf
```

### 4.1 日期规则

优先级：

1. 如果用户文件名中包含日期，优先使用文件名日期；
2. 如果用户明确指定日期，使用用户指定日期；
3. 如果没有，则使用论文发表日期、会议日期或版权页日期；
4. 如果仍无法确定，使用当前日期。

### 4.2 论文名规则

使用论文原始标题。

如果标题太长，可以合理简写，但不能改变原意。

### 4.3 括号内容规则

括号内格式：

```text
（关键词或主要内容，发表单位 作者，）
```

其中：

- 关键词或主要内容：写论文核心方法、对象或变量，尽量短；
- 发表单位：可使用期刊、会议、学校、研究机构或公司；
- 作者：作者太多时只写第一作者。

---

## 5. 首页文本框规则

### 5.1 首页文本框作用

首页文本框只作为“高密度文献速览卡”。

默认情况下，首页文本框内部**不做红色下划线和黄色高亮**。  
红线和黄标应尽量落在论文原文的具体位置。

### 5.2 首页文本框内容

文本框尽量包含以下字段：

```text
方法：
对象：
数据：
结论：
结果：
关键词：
注意：
```

也可以使用完整字段：

```text
方法：
研究对象：
实验/数据：
结论：
关键结果：
关键词：
注意点：
```

字段名称可以适度缩短，以节省空间。

### 5.3 方法

写论文使用的核心方法、实验流程、算法、模型或检测流程。

示例：

```text
方法：RGB→HSV，选取 S/Saturation 分量；用 single scale retinex 校正光照，再用 Xu segmentation 与 ICM 优化。
```

### 5.4 对象 / 研究对象

写论文研究的材料、缺陷、工艺、样品或检测对象。

示例：

```text
对象：protective coatings / painted surface 的 rusted areas 分割，并估计锈蚀百分比。
```

### 5.5 数据 / 实验

写样品数量、图像数量、测试标准、实验条件、分类等级、对比方法等关键信息。

示例：

```text
数据：不同锈蚀比例、不同对象、不同光照条件的测试图像；对比 classic ICM、Xu 分割和本文方法。
```

### 5.6 结论

写论文最核心的 2–4 条结论。

要求：

- 只写论文本身已经证明或明确说明的内容；
- 不加入项目应用分析；
- 不加入工业自动化方案；
- 不加入 AI 监控、AI 调参等延伸；
- 不把实验室结果夸大成工业现场结论。

### 5.7 结果 / 关键结果

写论文中的重要数值、准确率、性能指标、趋势、对比结果或实验现象。

如果论文没有明确数值，也可以写定性关键结果，但必须来自论文内容。

### 5.8 关键词

优先使用论文原始关键词。

如果论文原始关键词太少，可以从标题、摘要、方法和实验对象中补充。

关键词数量建议为 6–12 个。

### 5.9 注意 / 注意点

只写论文内部值得注意的限制、特殊条件、适用范围、不确定之处或方法风险。

可以包括：

- 数据规模较小；
- 图像采集条件较标准化；
- 方法依赖光照或颜色空间；
- 某算法可能陷入局部最小；
- 结果只在特定实验对象或条件下验证；
- 论文未给出大规模定量指标；
- 某些实验条件不能简单推广。

禁止写：

- “可用于 AI 自动调参”；
- “可用于工业闭环控制”；
- “适合作为产线监控模型”；
- 用户项目规划；
- 论文没有说过的推测。

## 6. 原文证据型标注规则

### 6.1 核心原则

本技能禁止简单按关键词全文重复标注。

原文标注必须采用“证据型标注”策略：

- 红色下划线：标关键概念的代表位置；
- 黄色高亮：标支撑首页速览卡的证据句；
- 每个关键词默认最多标 1 次；
- 只有特别重要的词，最多允许标 2 次；
- 标注目标是帮助用户快速找到论文依据，而不是把整页标满。

### 6.2 红色下划线规则

红色下划线只标注关键概念的首次有效出现或最有代表性的位置。

每个关键词在全文中默认最多标注 1 次。

如果一个关键词同时在“定义位置”和“结果位置”都非常重要，最多允许标注 2 次。

优先位置：

1. 标题中的核心研究对象；
2. 摘要中的首次方法出现；
3. 关键词区；
4. 方法章节中的定义位置；
5. 结果章节中的关键指标位置。

适合红线标注的内容：

- 论文核心方法；
- 算法名称；
- 模型名称；
- 材料名称；
- 关键变量；
- 检测对象；
- 重要标准；
- 性能指标名称。

示例：

```text
HSV Color Model
Saturation component
single scale retinex
Xu segmentation
ICM method
rust detection
rust estimation
ASTM B117
CNN
AlexNet
GoogleNet
ResNet-50
accuracy
precision
recall
F1 score
```

### 6.3 泛词过滤规则

以下类型的词不要单独反复标注：

```text
method
image
model
result
rust
segmentation
detection
processing
system
data
```

除非它们和核心概念组成明确术语，例如：

```text
HSV color model
rust detection
Xu segmentation
image processing
single scale retinex
decision support system
```

### 6.4 黄色高亮规则

黄色高亮只标注能支撑首页速览卡的证据句。

不要因为一句话里包含关键词就高亮。

优先高亮：

1. 研究目的句；
2. 方法流程句；
3. 实验数据或实验条件句；
4. 关键结果句；
5. 结论句；
6. 论文自身限制句。

适合黄标的句子类型：

```text
本文提出了什么方法。
为什么选择该方法。
实验使用了什么数据或条件。
结果证明了什么。
论文承认了什么限制。
```

不适合黄标：

```text
泛泛的背景介绍。
过长的整段文字。
只因为含有关键词的普通句子。
重复表达同一信息的句子。
```

### 6.5 每页标注数量限制

普通页面：

- 红色下划线最多 3–6 处；
- 黄色高亮最多 2–4 处。

第一页：

- 红色下划线最多 6 处；
- 黄色高亮最多 4 处。

短论文全文建议：

- 红色下划线总数：8–15 处；
- 黄色高亮总数：6–12 处。

普通长论文全文建议：

- 红色下划线总数：12–25 处；
- 黄色高亮总数：10–20 处。

如果标注后页面显得密集，必须减少标注数量。

### 6.6 原文标注优先级

如果时间或空间有限，优先标注这些位置：

1. Abstract / 摘要；
2. Keywords / 关键词；
3. Methods / Materials and Methods；
4. Experimental Results / Results；
5. Conclusion / Conclusions；
6. 重要图表标题和表格标题。

## 7. 首页文本框与原文标注的关系

首页文本框相当于“索引卡”。

原文标注相当于“证据位置”。

执行时应做到：

1. 文本框里写概括；
2. 原文里标出支撑这些概括的实际句子；
3. 关键词在原文首次有效出现或最代表性位置处红线；
4. 重要结论在原文出处位置黄标；
5. 不把所有标注都挤到文本框里；
6. 不在原文里重复标同一个词。

示例：

首页文本框写：

```text
方法：RGB→HSV，选取 S/Saturation 分量；用 single scale retinex 校正光照，再用 Xu segmentation 与 ICM 优化。
```

原文中应优先标注：

- 标题或摘要中的 `HSV Color Model`；
- 方法章节中的 `saturation component`；
- 方法章节中的 `single scale retinex`；
- 方法章节中的 `Xu segmentation`；
- 摘要或方法章节中的 `ICM method`；
- 摘要中的方法流程句；
- 结论中的最终结果句。

不要把 `HSV`、`segmentation`、`method` 在全文每次出现都标出来。

---

## 8. 信息量标准

首页文本框目标不是越短越好，而是让用户打开第一页就能复盘论文。

默认信息量：

- 短论文：约 180–280 中文字；
- 普通论文：约 250–450 中文字；
- 信息量大的论文：约 400–650 中文字。

默认行数：

- 不少于 6 行；
- 一般不超过 14 行；
- 若首页空间允许，可以稍长；
- 若首页空间不足，应优先压缩表达，而不是删到只剩关键词。

---

## 9. 首页布局自适应规则

### 9.1 放置区域优先级

第一页标注框优先放置在以下区域：

1. 标题上方的整块页眉空白区域；
2. 期刊 logo / 页眉下方但不遮挡标题的空白区域；
3. 标题下方但不遮挡作者和摘要的空白区域；
4. 摘要上方的空白区域；
5. 最后才考虑标题与作者之间的小空白区域。

不要优先把文本框塞进标题和作者之间，因为该区域通常高度不足，容易导致字体过小。

### 9.2 文本框尺寸

推荐样式：

- 宽度：页面宽度的 88%–92%；
- 左右边距：约 4%–6%；
- 内边距：4–6 pt；
- 背景：白色或轻微透明白色；
- 边框：红色或浅灰色细线；
- 字号：默认 8.5–10 pt；
- 最小字号：不得低于 8 pt；
- 行距：1.05–1.2；
- 文本框高度：根据内容自动计算，不固定成很矮的一条。

### 9.3 首页空间不足时的处理顺序

如果第一页顶部空间不足，按下面顺序处理：

1. 优先压缩“注意点”；
2. 再减少关键词数量；
3. 再把结论压缩为 2–3 条；
4. 再压缩实验/数据描述；
5. 最后才允许小幅减小字号，但不得低于 8 pt。

### 9.4 允许的首页轻微重排

如果不轻微重排就无法保证可读性，允许只对第一页做以下处理：

1. 将第一页原文整体等比例缩小；
2. 将第一页原文整体向下移动；
3. 在顶部腾出正常可读的标注框空间；
4. 后续页面保持不变；
5. 缩放不能导致原文明显不可读；
6. 不能删减、遮挡或覆盖原论文内容。

## 10. 执行流程

### Step 1：读取 PDF

读取：

- 首页；
- 摘要；
- 关键词；
- 方法；
- 实验/数据；
- 结果；
- 结论。

短论文应尽量阅读全文。

长论文至少读取摘要、方法、数据、结果表格、结论部分。

### Step 2：提取命名信息

提取：

- 日期；
- 论文标题；
- 关键词或主要内容；
- 发表单位；
- 第一作者。

生成新文件名。

### Step 3：生成首页速览卡

生成字段：

```text
方法：
对象：
数据：
结论：
结果：
关键词：
注意：
```

内容应紧凑、准确、可复盘。

### Step 4：生成证据型标注清单

从速览卡中反向生成原文标注清单：

1. 核心概念清单：用于红色下划线；
2. 证据句清单：用于黄色高亮；
3. 证据位置：优先在摘要、关键词、方法、结果、结论中寻找。

生成清单时必须去重：

- 同一关键词默认只保留 1 个最代表位置；
- 同一证据含义只保留 1 句最清楚的句子；
- 删除泛词和重复词。

### Step 5：定位原文位置

在 PDF 原文中搜索关键词和证据句。

优先使用 PDF 内部文本坐标定位。

如果 PDF 文本层可用：

- 用文本搜索定位词语；
- 在词语坐标处画红色下划线；
- 在句子坐标范围画黄色高亮。

如果 PDF 文本层不可用或定位失败：

- 可以使用页面渲染图辅助定位；
- 无法精确定位的标注应跳过；
- 不要把原文标注伪造到文本框里。

### Step 6：绘制首页文本框

在第一页顶部绘制速览卡。

默认文本框内不做红色下划线和黄色高亮。

### Step 7：绘制原文证据型标注

在论文实际位置绘制：

- 红色下划线：关键概念；
- 黄色高亮：证据句。

控制数量，避免整页密集标注。

### Step 8：输出文件

保存为：

```text
日期-论文名（关键词或主要内容，发表单位 作者，）_已标注.pdf
```

---

## 11. 禁止事项

执行本技能时禁止：

1. 新增文献标注页；
2. 把命名信息单独写进 PDF 标注框；
3. 把代号写进文件名；
4. 只在文本框里画红线和黄标，而不尝试标注原文；
5. 按关键词全文重复标注；
6. 每次出现同一个词都画线；
7. 把泛词当成重点反复标；
8. 黄标整段背景介绍；
9. 写用户项目应用分析；
10. 写工业自动化方案；
11. 写 AI 产线监控方案；
12. 写自动调参方案；
13. 写论文没有出现的推测；
14. 覆盖原 PDF 正文；
15. 删除原 PDF 内容；
16. 让字体小到不可读；
17. 把文献标注另存为 Word 文档，除非用户明确要求。

## 12. 默认输出示例

首页文本框示例：

```text
方法：RGB→HSV，选取 S/Saturation 分量；用 single scale retinex 校正光照，再用 Xu segmentation 与 ICM 优化。
对象：protective coatings / painted surface 的 rusted areas 分割，并估计锈蚀百分比。
数据：不同锈蚀比例、不同对象、不同光照条件的测试图像；对比 classic ICM、Xu 分割和本文方法。
结论：S 分量比 H 更适合；retinex 可降低光照影响；Xu 初分割 + ICM 可进一步补全相邻锈蚀区域。
结果：Fig.2–5 展示原图、classic ICM、Xu segmentation、proposed method 对比；未给出大规模定量准确率。
关键词：HSV；Saturation；single scale retinex；Xu segmentation；ICM；rust detection；segmentation。
注意：ICM 可能停在局部最小；初始分割质量会影响最终结果；未知涂层颜色和照明可能影响分割。
```

原文标注示例：

红线只标：

- 标题中的 `HSV Color Model`；
- 摘要或方法中的 `single scale retinex`；
- 方法中的 `saturation component`；
- 方法或摘要中的 `ICM method`；
- 关键词区中的 `rust detection`；
- 关键词区中的 `segmentation`。

黄标只标：

- 摘要中说明研究问题的句子；
- 摘要中说明方法流程的句子；
- 摘要中说明结果可用于 rust estimation 的句子；
- 方法章节中说明为什么选 S 分量的句子；
- 实验结果中列出 1–4 步流程的句子；
- 结论中说明 proposed method 能 distinguish rust regions 的句子。

## 13. 适用范围

适合：

- 电镀论文；
- 表面处理论文；
- 腐蚀检测论文；
- 材料论文；
- 化工工艺论文；
- 制造业实验论文；
- 图像检测、缺陷检测、质量分级类论文；
- 需要直接在 PDF 首页做高密度速览，并在原文中定位重点证据的文献。

不适合：

- 需要长篇综述的任务；
- 需要独立 Word 总结的任务；
- 需要用户项目方案分析的任务；
- 需要重新排版整篇 PDF 的任务；
- 需要对论文内容做大量个人推测的任务。
